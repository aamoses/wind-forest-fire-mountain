# 风林火山·四方乱斗 — 最终设计文档

> 版本：v3.0  
> 日期：2026-05-17  
> 类型：四人平板围坐 · 回合制策略对战

---

## 一、游戏概述

### 1.1 核心概念

《风林火山·四方乱斗》是一款四人本地对战的回合制策略游戏。玩家围坐在平放的平板电脑四周，每人控制一个阵营，按顺时针方向轮流行动。游戏取材自《孙子兵法》"其疾如风，其徐如林，侵掠如火，不动如山"，四个阵营各具独特的战斗风格。

### 1.2 目标平台

- 平板电脑（横屏优先，10英寸以上最佳）
- 桌面浏览器，支持触屏与鼠标
- 纯 HTML5/CSS3/JavaScript，无第三方框架

### 1.3 设计理念

**视觉关键词**：暗色奢华 · 金色光效 · 3D 麻将俯视视角 · 微缩战场

整个游戏呈现为一张置于深色桌面上的立体棋盘，棋子如同雀龙门麻将牌般有厚度感，金色粒子在战场上空浮动。四边信息栏环绕棋盘，当前行动方由梯形光环标识。

## 二、棋盘与点位

### 2.1 棋盘规格

棋盘由 **33 个交叉点** 构成，布局在 8 列 × 7 行的网格上，划分为 5 个区域。相邻点之间由 **52 条边** 连接。

### 2.2 点位坐标与区域划分

```
列(col):  0  1  2  3  4  5  6  7
行(row)
  0                  9 10 11          ← 上方田字格（火阵营，zone: top）
  1                 12 13 14
  2        21 22     0  1  2    27 28
  3        23 24     3  4  5    29 30   ← 中央田字格（zone: center）
  4        25 26     6  7  8    31 32
  5                 15 16 17          ← 下方田字格（山阵营，zone: bottom）
  6                 18 19 20
         ↑          ↑       ↑
    左方田字格    中央     右方田字格
  （林阵营，     (center) （风阵营，
   zone: left）           zone: right）
```

| 区域 | zone | 点位 ID | 格数 | 归属阵营 |
|------|------|---------|------|----------|
| 中央 | `center` | 0-8 | 3×3 = 9 | 中立 |
| 上方 | `top` | 9-14 | 3×2 = 6 | 火 (fire) |
| 下方 | `bottom` | 15-20 | 3×2 = 6 | 山 (mountain) |
| 左方 | `left` | 21-26 | 2×3 = 6 | 林 (forest) |
| 右方 | `right` | 27-32 | 2×3 = 6 | 风 (wind) |

### 2.3 相邻关系

共 52 条边，每个中心区域点位有 4 个邻居，边缘点位有 2-3 个邻居。相邻关系用于移动、近战攻击判定和技能范围计算。

## 三、阵营系统

### 3.1 阵营属性

| 属性 | 火 (Fire) | 林 (Forest) | 风 (Wind) | 山 (Mountain) |
|------|-----------|-------------|-----------|---------------|
| **口号** | 侵掠如火 | 其徐如林 | 其疾如风 | 不动如山 |
| **主色** | `#e74c3c` 赤红 | `#27ae60` 翠绿 | `#f1c40f` 金色 | `#2980b9` 冰蓝 |
| **棋子色** | `#ff3b20` | `#38e868` | `#f5d040` | `#4da8ff` |
| **基地色** | `#3a0e08` | `#062a0e` | `#2e1c04` | `#051a32` |
| **阵地区** | 上方 (top) | 左方 (left) | 右方 (right) | 下方 (bottom) |

### 3.2 克制关系

双向克制链，形成闭环：

```
火 ──(克 1.25×)──→ 风 ──(克 1.25×)──→ 林 ──(克 1.25×)──→ 山 ──(克 1.25×)──→ 火
  ←──(被克 0.75×)──   ←──(被克 0.75×)──   ←──(被克 0.75×)──   ←──(被克 0.75×)──
```

| 攻击方 \ 防御方 | 火 | 林 | 风 | 山 |
|-----------------|-----|-----|-----|------|
| 火 | 1.0 | 1.0 | **1.25** | 0.75 |
| 林 | 1.0 | 1.0 | 0.75 | **1.25** |
| 风 | 0.75 | **1.25** | 1.0 | 1.0 |
| 山 | **1.25** | 0.75 | 1.0 | 1.0 |

### 3.3 阵营技能

#### 火 — 火焰贯穿（直线穿透）

- **触发**：选择棋子后点击「技能」，出现四个方向箭头（↑↓←→）
- **效果**：火焰沿选定方向直线飞行，命中路径上首个敌方棋子
- **射程**：无限（直至棋盘边缘或命中棋子）
- **伤害**：2 点基础伤害（受克制系数影响）
- **特效**：红色方向箭头 → 火焰光线动画 → 目标点爆发闪光

#### 林 — 陷阱布设与引爆

**布陷阱：**
- **触发**：选择棋子后点击「布陷阱」，点击空位放置
- **效果**：在空位埋设隐藏陷阱
- **可视性**：只有林阵营玩家能看到己方陷阱（绿色），其他阵营不可见
- **特效**：目标点出现绿色光圈

**引爆：**
- **触发**：点击「引爆」，再点击可见的绿色陷阱
- **范围**：陷阱所在点 + 所有相邻点（最多 9 格 AOE）
- **伤害**：范围内每个非林阵营棋子受 2 点伤害
- **免疫**：林阵营棋子不受陷阱伤害
- **特效**：陷阱点紫色爆发 → 向四周扩散波纹

#### 风 — 风刃（日字 AOE）

- **触发**：选择棋子后点击「技能」→「确认释放风刃」
- **范围**：以棋子为中心的"马走日" 8 个点位（±1 列 ±2 行，±2 列 ±1 行）
- **伤害**：范围内**所有棋子**（含友军）各受 2 点伤害
- **警告**：风是唯一会误伤友军的技能
- **特效**：多点金色闪光同时爆发

#### 山 — 走打一体

- **触发**：选择棋子后自动进入走打模式
- **步数池**：每回合 6 步
- **移动**：每移动一格（相邻）消耗 1 步
- **攻击**：相邻敌方 → 消耗**全部剩余步数**作为伤害值
- **限制**：不可中途换棋子操作（除非步数耗尽或手动结束回合）
- **特效**：步数计数器（6 个蓝色圆点，已用者变暗缩小）

| 每棋子初始 HP | 4 点 |

| 棋子尺寸 | 56×64 px（含底座 + HP 指示器） |

### 4.2 初始布阵

每阵营 6 个棋子部署在各自田字格内，具体点位如下：

| 阵营 | 初始点位 |
|------|----------|
| 火 (fire) | 9, 10, 11, 12, 13, 14 |
| 山 (mountain) | 15, 16, 17, 18, 19, 20 |
| 林 (forest) | 21, 22, 23, 24, 25, 26 |
| 风 (wind) | 27, 28, 29, 30, 31, 32 |

### 4.3 淘汰判定

- 阵营所有棋子 HP ≤ 0 → 该阵营被淘汰
- 淘汰判定在每次攻击/技能后自动检查
- 当场上仅剩 1 个阵营时 → 游戏结束，该阵营获胜

## 五、规则详解

### 5.1 回合流程

```
回合开始 → 当前阵营行动 → 检查淘汰 → 顺时针移至下一阵营 → 重复
```

### 5.2 行动顺序

1. 四个阵营随机分配到平板的四条边（上/右/下/左）
2. 随机决定起始边（0-3）
3. 按顺时针方向轮流行动：上 → 右 → 下 → 左 → 上 → …
4. 围绕平板的四名玩家各坐一边，正对自己的方向就是自己的阵营

### 5.3 布阵阶段

- 前 **8 回合**（2 轮完整循环）为布阵阶段
- 布阵期间：可以**移动**和**布陷阱**，但**禁止一切攻击行为**
- 目的：让各方有时间展开阵型进入有利位置
- 提示文字：「前两轮为布阵阶段，不可攻击」

### 5.4 移动规则

- 每次移动只能走到**相邻空位**
- 不能移动到有棋子（己方或敌方）占据的点位
- 移动后自动结束回合

### 5.5 近战攻击规则

- 范围：相邻 1 格内的**敌方棋子**
- 伤害：2 点基础伤害（受克制影响）
- 效果：攻击者**占据防御者的位置**（位移攻击）
- **无区域限制**：在己方领地、敌方领地、中央区域均可攻击
- **安全区例外**：在己方初始领地内时，防御方有优势，己方不能主动攻击

### 5.6 技能使用规则

- 技能与近战攻击一样受布阵阶段限制
- 林阵营布陷阱不受布阵阶段限制（可在布阵期布置）
- 火技能可穿越空位，但遇到第一个棋子即停止
- 风技能波及范围内**全部棋子**（含友军），不可选择
- 山技能不可中途换棋子（除非步数耗尽）

## 六、游戏模式

### 6.1 单人修习（Single）

| 项目 | 详情 |
|------|------|
| 人类玩家 | 1 人（固定下方位置） |
| AI 对手 | 3 个（上方、右方、左方） |
| 阵营分配 | 随机 |
| 操作方式 | 触屏点击下方棋子 |

### 6.2 双人组阵（Dual）

| 项目 | 详情 |
|------|------|
| 人类玩家 | 2 人（下方 + 右方，相邻而坐） |
| AI 对手 | 2 个（上方、左方） |
| 阵营分配 | 随机 |
| 协作方式 | 两人阵营相邻，自然形成战线联动 |

### 6.3 四方会战（Quad）

| 项目 | 详情 |
|------|------|
| 人类玩家 | 4 人（每条边各一人） |
| AI 对手 | 无 |
| 阵营分配 | 随机 |
| 操作方式 | 各自操作面对自己的那方棋子 |

### 6.4 模式选择界面

主菜单以暗色奢华风格呈现：
- 顶部：金色 ⚔ 徽标（浮动动画，±6px）
- 标题「风林火山」（48px，金色发光字，12px 字间距）
- 副标题「四方乱斗」
- 金色渐变分隔线
- 提示文字「阵营随机分配 · 顺时针行动」
- 三个模式按钮（带图标、标题和描述文字）
- 底部内联战绩面板：段位徽章、进度条、总场次/胜率/最佳阵营

### 7.2 背景设计

**页面背景：**
- 暖调深棕黑渐变：中心偏琥珀色 → 边缘纯黑
- 叠加两层径向渐变模拟桌面光泽
- 伪元素 `body::before` 在中央投射大范围微弱金光

**画布粒子（Canvas）：**
- 150 个浮动粒子，4 种类型：

| 类型 | 颜色 | 大小 | 行为 | 占比 |
|------|------|------|------|------|
| 金色微尘 | `rgb(240,210,96)` | 0.3-1.7px | 缓慢飘浮 | 45% |
| 暗红火星 | `rgb(255,100,40)` | 0.6-2.6px | 快速上升 | 25% |
| 蓝白冷光 | `rgb(100,180,255)` | 1-3.5px | 缓慢飘移 | 18% |
| 大型光斑 | `rgb(240,210,80)` | 2-6px | 极慢飘浮 | 12% |

- 所有粒子：正弦波脉动透明度 + 径向渐变光晕 + 边界循环
- 60fps requestAnimationFrame 驱动

### 7.3 棋盘视觉

**棋盘容器：**
```
尺寸：532 × 460 px
视角：perspective(900px) + rotateX(18deg) → 2.5D 俯视
背景：深胡桃木色径向渐变（中心偏亮）
边框：2px 金色实线
阴影：四层叠加
  - 0 0 0 1px 暗金边
  - 0 0 0 8px 深色外框
  - 0 0 50px 微弱金光
  - 0 0 120px 黑色投影
  - inset 0 0 100px 微妙内发光
```

**网格线（SVG）：**
- 线条颜色：`rgba(180,155,100,0.35)`
- 宽度：1.5px
- 交点：圆点 5px 半径，`rgba(180,155,100,0.28)`

**中央区域：**
- 金色径向渐变柔光
- 缓慢明暗交替动画（5 秒周期）

**四角金饰：**
- 14×14 px 金色渐变方块
- 闪烁动画（2 秒周期交替）

### 7.4 回合指示器（梯形光环）

位于棋盘外围的梯形裁切光环：

- 位置：棋盘外围 +12px
- 形状：clip-path 梯形多边形，**宽边朝向当前行动玩家**
- 过渡：0.55 秒平滑切换
- 内环：2px 金色边框 + 光晕
- 外环：1px 亮金边框 + 脉冲动画（2.5 秒周期）
- 宽边附加光扫效果（2 秒左右扫动）

### 7.5 3D 棋子设计

#### 底座（.piece-base）

```
尺寸：44×44 px 圆形
3D 效果：
  - 底部投影：0 4px 0 1px 深色 → 0 5px 0 3px 更深 → 0 8px 16px 黑
  - 顶面高光：radial-gradient（白色 12% → 透明 60%）
  - 侧面渐变：linear-gradient（顶亮 → 底暗）
  - 阵营色边框 2px
  - 阵营色外发光
```

#### 兵人形象（.piece-figure）

每个棋子嵌入 SVG 兵人，具体造型见第八节。SVG 嵌入方式：
```html
<div class="piece-figure" style="transform:rotate(Xdeg);">
  <!-- SVG 兵人 -->
</div>
```

#### 朝向（.piece-figure transform）

兵人面朝操控该阵营的玩家。SVG 默认面朝上方（屏幕顶部），需旋转：

| 玩家位置 | 旋转角度 | 说明 |
|----------|----------|------|
| 上方 (top) | 0° | 兵人本就面向上方玩家 |
| 下方 (bottom) | 180° | 兵人面转向下方玩家 |
| 左方 (left) | 90° | 兵人面转向左方玩家 |
| 右方 (right) | 270° | 兵人面转向右方玩家 |

#### 生命指示器（.piece-hp）

- 4 个 5×5 px 圆点并排
- 存活 HP：阵营色发光圆点
- 损失 HP：暗淡缩小，透明度 0.15

## 八、SVG 兵人详细设计

每个兵人 SVG 采用 `48×56` viewBox，包含以下元素层次：

### 8.1 共同结构

1. **底部投影**：深色椭圆 `rgba(0,0,0,0.5)`
2. **双腿**：两个矩形，阵营暗色
3. **躯干**：梯形或多边形，阵营主色
4. **胸部装甲**：矩形或路径，阵营辅色
5. **头部**：圆形肤色面庞 + 头盔/兜帽
6. **阵营光晕**：径向渐变圆，颜色与阵营匹配，带 `<animate>` 呼吸效果

### 8.2 火阵营兵人（枪兵）

| 元素 | 规格 |
|------|------|
| 头盔 | 棕色梯形盔顶饰 |
| 躯干 | 暗红梯形 ×2 层装甲 |
| 武器 | 长矛 — 金色竖线 + 金色三角矛头 |
| 待机动画 | 矛尖火焰粒子圆 2-4px 脉冲（0.8 秒） |
| 核心光晕 | 红色渐变圆 4-6px 呼吸（1.5 秒） |
| 姿态 | 正面站立，矛持右侧 |

### 8.3 林阵营兵人（弓兵）

| 元素 | 规格 |
|------|------|
| 兜帽 | 绿色弧形路径覆盖头部 |
| 躯干 | 绿色矩形 + 暗绿肩甲 |
| 武器 | 长弓 — 棕色弧线 + 金色箭矢 |
| 待机动画 | 无（静态，突出沉着冷静） |
| 核心光晕 | 绿色渐变圆 3-5px 呼吸（2 秒） |
| 姿态 | 正面站立，弓横持 |

### 8.4 风阵营兵人（斥候）

| 元素 | 规格 |
|------|------|

## 十一、AI 引擎设计

### 11.1 决策算法

**贪心搜索 + 概率扰动**：
1. 遍历当前阵营所有存活棋子
2. 为每个棋子生成所有合法操作（移动/近战/技能）
3. 对每个操作打分
4. 按分数降序排列
5. 70% 概率选最优，30% 概率在前 3 名中随机选

### 11.2 评分函数

| 操作类型 | 评分规则 |
|----------|----------|
| **移动** | 基础 5 分 + 距中心越近越高（每格 +1.5）+ 进入敌方领地 +10 |
| **近战** | 伤害 ×8 + 击杀奖励 +30 + 克制优势 +10 / 克制劣势 -5 |
| **火技能** | 伤害 ×7 + 击杀奖励 +25 + 克制优势 +8 |
| **风技能** | 总伤害 ×5 + 命中数 ≥2 额外 +8 + 每友军误伤 -10 |
| **布陷阱** | 2 格内每有 1 个敌人 +8 |
| **引爆** | 每命中 1 个敌人 +12 |
| **山技能** | 伤害 ×7 + 剩余步数 ≥3 额外 +5 + 击杀 +25 |

### 11.3 AI 行为特征

| 行为 | 描述 |
|------|------|
| **思考延迟** | 120-350ms 随机延迟，模拟人类反应 |
| **心理提示** | 执行前显示「{阵营名}思考中...」 |
| **领地意识** | 优先向敌方领地推进，避免退回己方领地 |
| **中心争夺** | 倾向占据中央区域（战略要地） |
| **陷阱策略** | 在敌方棋子 2 格范围内的要道布设陷阱 |
| **引爆时机** | 优先引爆能命中 2+ 敌人的陷阱 |
| **山阵营特殊处理** | 搜索 1-2 步移动后攻击的最优路径 |

### 11.4 AI 触发时机

```
非部署阶段 → 当前回合是AI阵营 → 延迟120-350ms → 执行AI回合
```

人类的操作锁定：
- 点击棋子时检查 `gameState.humanFactions.includes(currentFaction())`
- AI 回合中，人类无法点击任何己方棋子
- AI 回合中操作按钮不显示

## 十六、设计愿景总结

### 玩家体验目标

1. **围坐平板的仪式感**：四人各坐一边，阵营随机分配，棋子面朝自己，梯形光环宽边指向自己——所有这些细节共同营造"我们围坐在这张虚拟战桌旁"的沉浸感。

2. **金色暗黑美学**：不是花哨的卡通风格，而是沉稳、奢华、有分量的"国际赛事"质感。深胡桃木色为底，金色为唯一强调色，粒子和光晕都在金-琥珀-暗红范围内。

3. **棋子是活的**：每个棋子不是静态 emoji，而是一个微型战场雕像——SVG 兵人有盔甲有武器，有待机呼吸动画，被选中时抬升发光，移动时有平滑过渡，攻击时目标点爆发光效。如同自走棋的棋子一般，有自己的存在感。

4. **信息四向分布**：每个玩家看到的信息栏是正对自己的，文字方向旋转适配。当前谁的回合一目了然——不仅是信息栏高亮，梯形光环的宽边也指向你。

5. **策略深度**：四阵营非对称设计，火贯穿、林陷阱、风 AOE（含友伤）、山走打一体，加上克制链和领地安全区，产生丰富的战术博弈空间。
